Form widgets

Although we designed these widgets for stupid.css, they will work with any CSS library. They are designed to be simple to use and yet powerful by inferring the proper behaviour from the HTML tags. They work great with Bootstrap and Metarialize for example. The size of "widgets.min.js" is ~6KB.

Instructions

All you need to do it include the following:

<script src="jquery.js"></script>
                <script src="sugar.min.js"></script>
                <script src="widgets.min.js"></script>

And then use the proper classes and attributes to apply the widgets.

Integer Widget

You need is a class .integer:

<input class="integer"/>

The widget will prevent you from trying invalid numbers in the field.

Optionally you can specify a data-range:

<input class="integer" data-range="0,100"/>

If the number is outside bounds, an .error class is applied.

Notice you can use a data-range also for numbers, dates, times, and strings.

Number Widget

All you need is a class .number:

<input class="number"/>

The widget will prevent you from tying invalid numbers in the field.

Optionally you can specify a data-range:

<input class="number" data-range="0,100"/>

If the number is outside bounds, an .error class is applied

Slider Widget

You need is a class .slider applied to an .integer or a number field and a data-range attribute:

<input class="number slider" data-range="0,100,1"/>

It will display a slider on top of the input field.

It automatically figures out the precision (number of digits) from the range.

The third argument of the data-range is an optional step.

Date/Time/Datetime Widget

This widget requires "sugar.js".

To activate the widget you need one of the following classes: .date, .time, .datetime(does not matter which one) and a data-format attribute (very important). The format must be in the "sugar.js" format. For example:

<input class="date" data-format="{dd}/{MM}/{year}"/>
<input class="time" data-format="{HH}:{mm}:{ss} {TT}"/>
<input class="datetime" data-format="{dd}/{MM}/{year} {HH}:{mm}:{ss} {tt}"/>

Notice that despite the arbitrary representation for date/time/datetime, the actual value of the field is always expected to be in ISO format and when the form is submitted, it will be submitted in ISO format. If you choose to specify a data-range, the values of the range should also be in ISO format. The idea of the widget is to complete decouple the UI represnetation with the server side handling (which should always be in ISO format). To internationalize the widet set the locale using "sugar.js".

Regex Validation

You can specify a regex validator for any field (including dates) with a data-regex. For example you can specify a 5 digits zip code:

<input data-regex="^(\d{5})?$"/>

For Angular, React, and Ractive.js users

If you use library that manipulates the DOM, the calendar and slider may break. All you have to do it re-apply them when new input elements are injected in the DOM:

jQuery('.date,.datetime,.time').calendar();
                jQuery('.slider').slider();

These functions take no argments.

What about styling?

They are easy to style with regular CSS. Should work with any CSS framework without too much trouble.
